## Summary

Deploy ACSC Edge hardening configuration through Intune to enforce the Essential Eight Edge controls across devices. The implementation aligns with ASD and vendor hardening guidance, applying the most restrictive settings when conflicts occur. Edge hardening is implemented via Intune using the ACSC Edge Hardening Guidelines and the Edge Security Baseline.[^1]

## Implementation Steps

### Import ACSC Edge Hardening Guidelines policy

1. Save the ACSC Edge Hardening Guidelines policy to your local device. [ACSC Edge Hardening Guidelines.json](https://github.com/microsoft/Intune-ACSC-Windows-Hardening-Guidelines/blob/main/edge/policies/ACSC%20Edge%20Hardening%20Guidelines.json)  
2. Open the Microsoft Intune admin center.  
3. Import a policy: Devices > Windows > Configuration profiles > Create > Import Policy.  
4. Name the policy, select Browse for files under Policy file, and navigate to the saved policy from step 1.  
5. Save. [^1]

### Deploy Edge Hardening using Security Baseline

1. Navigate to Endpoint Security > Security Baselines > Microsoft Edge Baseline.  
2. Create a new Microsoft Edge Baseline by selecting Create Profile.  
3. Review the configuration, and assign the Security Baseline to a group. [^1]
