## Summary

Local Security Authority (LSA) protection isolates the LSASS process to guard credentials from untrusted code.[^1] Enable added LSA protection via Intune by creating a custom device configuration profile and configuring the OMA-URI setting ConfigureLsaProtectedProcess to 1 (with UEFI lock) or 2 (without UEFI lock), followed by a reboot.[^1] Intune security baselines provide default Windows security configurations that facilitate deploying LSA protection at scale.[^2]

[^1]:Configure added LSA protection | Microsoft Learn(https://learn.microsoft.com/en-us/windows-server/security/credentials-protection-and-management/configuring-additional-lsa-protection)
[^2]:Default configuration of Intune's Windows security baselines - Microsoft Intune | Microsoft Learn(https://learn.microsoft.com/en-us/intune/intune-service/protect/security-baseline-settings-mdm-all)

## Design Decision

> [!NOTE] Enable added LSA protection on devices by deploying an Intune custom configuration that sets the Local Security Authority to run LSASS as a protected process (ConfigureLsaProtectedProcess), or apply a security baseline that enforces LSA protection.

## Implementation Steps

### Enable LSA protection by using the registry (single computer)

1. Open the Registry Editor (RegEdit.exe).
2. Navigate to the following key:
   HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Lsa
3. Open the RunAsPPL value and set its data as:
   - To configure the feature with a UEFI variable: 00000001 (DWORD).
   - To configure the feature without a UEFI variable: 00000002 (DWORD; enforced on Windows 11 version 22H2 and later).
4. Restart the computer. [^1]

### Enable LSA protection by using Group Policy

1. Open the Group Policy Management Console by entering gpmc.msc in the Run dialog.
2. Create a new Group Policy Object (GPO) linked at the domain level or to the OU with the computer accounts, or select an existing one.
3. Right-click the GPO and select Edit to open the Group Policy Management Editor.
4. Expand Computer Configuration > Preferences > Windows Settings.
5. Right-click Registry, point to New, and select Registry Item.
6. In New Registry Properties, configure:
   - Hive: HKEY_LOCAL_MACHINE
   - Key Path: SYSTEM\CurrentControlSet\Control\Lsa
   - Value name: RunAsPPL
   - Value type: REG_DWORD
   - Value data: 00000001 (enable with a UEFI variable) or 00000002 (enable without a UEFI variable; enforceable on Windows 11 22H2+)
7. Select OK. [^1]
8. Note: To take effect, the GPO change must replicate to all domain controllers in the domain. [^1]

### Enable LSA protection by creating a custom device configuration profile (Intune)

1. In the Intune admin center, go to Devices > Windows > Configuration profiles, and select Create profile.
2. On Create profile, set:
   - Platform: Windows 10 and later
   - Profile type: Templates > Custom
   - Click Create
3. On Basics, enter a name and optional description, then select Next.
4. On Configuration settings, select Add.
5. On Add row, configure:
   - Name: (OMA-URI setting name)
   - OMA-URI: ./Device/Vendor/MSFT/Policy/Config/LocalSecurityAuthority/ConfigureLsaProtectedProcess
   - Data type: Integer
   - Value: 1 (LSA protected with UEFI lock) or 2 (LSA protected without UEFI lock)
6. Select Save, then Next.
7. On Assignments, configure assignments, then select Next.
8. On Applicability Rules, configure rules, then select Next.
9. On Review + create, verify configuration, then select Create.
10. Restart the computer. For more information about this CSP, see LocalSecurityAuthority - ConfigureLsaProtectedProcess. This approach aligns with Intune security baselines. [^1][^2]
