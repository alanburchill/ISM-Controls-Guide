## Summary

Browser security settings are enforced through Intune to prevent users from modifying browser configurations.[^1] Edge can be managed via Intune app protection policies and device configuration settings to restrict browsing behavior, including allowed sites and data handling.[^1][^3] This approach applies across iOS, Android, and Windows devices, aligning with Intune for Education and Android device restriction guidance to maintain a consistent security posture.[^4][^5]

[^5]:Device restriction settings for Android DA in Microsoft Intune(https://learn.microsoft.com/en-us/intune/intune-service/configuration/device-restrictions-android)

## Design Decision

> [!NOTE] Enforce browser security settings via Intune so users cannot modify them.

## Prerequisites

* **Licensing:** Not provided in source documentation.

## Implementation Steps

### Enforce Edge browser security on iOS and Android with Intune app protection policies

1. Sign into the Intune admin center.[^1]
2. Create Intune app protection policies for Edge for iOS and Edge for Android.[^1]
3. Set the data protection level to Enterprise Enhanced Data Protection (Level 2).[^1]
4. Configure the policy setting “Restrict web content transfer with other apps” to restrict transfers to Microsoft Edge.[^1]
5. Assign the policies to all users and devices.[^1]
6. For Android devices that aren’t enrolled in Intune, require the Intune Company Portal to apply the policy.[^1]

### Enforce Edge app configuration policies to manage the browsing experience

1. Sign into the Intune admin center.  
2. Create a Managed Apps App Configuration Policy for Edge (iOS and Android).[^2]
3. Configure the Edge browsing experience using the following keys and example values:
   - `com.microsoft.intune.mam.managedbrowser.NewTabPageLayout` = `custom` 
   - `com.microsoft.intune.mam.managedbrowser.NewTabPageLayout.Custom` = `topsites`
   - `com.microsoft.intune.mam.managedbrowser.NewTabPageLayout.UserSelectable` = `false`
   - `com.microsoft.intune.mam.managedbrowser.NewTabPage.BrandLogo` = `true`
   - `com.microsoft.intune.mam.managedbrowser.NewTabPage.BrandColor` = `true`
   - `com.microsoft.intune.mam.managedbrowser.homepage` = a valid URL (e.g., `https://www.contoso.com`)
   - `com.microsoft.intune.mam.managedbrowser.managedTopSites` = a pipe-delimited set of URL/Title pairs (e.g., `GitHub|https://github.com||LinkedIn|https://www.linkedin.com`)
   - `com.microsoft.intune.mam.managedbrowser.NewTabPage.IndustryNews` = `true`
   - `com.microsoft.intune.mam.managedbrowser.NewTabPage.CustomURL` = a valid URL (if using a custom URL)
   Note: These keys enable centralized control of New Tab Page layout, branding, homepage, and top-site shortcuts. See guidance for additional keys and formats in the source documentation.[^2]
4. Save and assign the policy to groups.  
5. Validate that users cannot override these settings when Edge is managed by Intune.[^2]

### Enforce browser usage through Conditional Access

1. Create a Conditional Access policy in the Microsoft Entra (Azure AD) portal.
2. Require approved client apps or an app protection policy for access to corporate resources via Microsoft Edge on iOS and Android.  
3. Target all users and all devices that access Microsoft 365 endpoints.  
4. Enforce the policy to ensure corporate data only flows through the managed Edge browser.[^1]

### Validate and verify enforcement

1. On a test device, confirm Edge is governed by Intune policies and that changes to configured settings are blocked.[^1]
2. Verify that data transfer is restricted to Edge and that non-Edge apps cannot access corporate data due to app protection settings.  
3. Confirm Conditional Access behavior blocks access when Edge is not the approved client or policy-compliant Edge usage is not met.[^1]
